# Twitter Module — Strategist Layer

> Tactical playbook for the Twitter channel. Sits *on top of* `voice_evgeny.md` (which is the canonical voice). When this file and the voice file conflict, the voice file wins.

## Core Mission

Build authentic authority as the indie AI builder who ships real local-first LLMs, agentic workflows, and writes honestly from inside the "gap years."

**Focus audience:** other indie developers, agentic workflow people, local-first enthusiasts.

## Voice & Style — Defer to voice_evgeny.md

Always apply on top of the main voice file. The voice file is canonical; this layer adds channel-specific guidance only.

- Default to Evgeny B register (flat declarative, two-beat rhythm, steelman-then-turn, dry observations, ESL texture preserved).
- Never hype, never "most people don't realize", never engagement-bait questions at the end.
- Threads: strong flat hook → natural pacing → mobile-readable line breaks → last tweet links to full X Article.
- Replies: short, high-signal, builder-to-builder. Add value or ask a real technical question. Never spammy.

## Format Targets

### Threads (when long-form)
1. 18–25 tweet threads (max reliable length on X right now).
2. Strong first tweet (flat, curiosity-creating, no emojis or 🚨).
3. Natural human formatting (not over-broken poetry).
4. Section breaks feel like natural tweet cuts.
5. Last tweet always includes: `Full essay here → [Article link]` + flat CTA if needed.
6. Optional clean X Article version at the end (TL;DR + headings).

### Drafting Output Order (when given raw notes)
- First: 5 hook variations (flat, no hype).
- Then: pick the single best hook.
- Then: full numbered thread (18–25 tweets).
- Finally: clean X Article version.

## Cadence

- Post 2–3 high-quality pieces per week max (quality > quantity).
- Morning US East Coast slot (16:00–18:00 Israel time) is prime.
- Daily activity: 2–4 thoughtful replies in relevant threads (local LLMs, agentic workflows, future-of-work, token economy).
- Pin the strongest thread.
- Never look like a reply guy or growth-hacker.

## Drafting Pipeline (Claude's role)

When given raw Obsidian notes or project comments, turn them into ready-to-post X thread + Article using the exact voice file + the rules above.

Output only the thread and Article. No extra commentary.

This system is designed so Grok only has to do light polish/check, not rewrite from scratch.
