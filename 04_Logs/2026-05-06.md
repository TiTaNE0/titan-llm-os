## Decisions
- **Voice file inside Content scope, not at vault root.** Voice belongs to content, not the OS kernel.
- **No version in filename.** `voice_evgeny.md` instead of `voice_evgeny_v1.3.md`. New versions overwrite in place; references stay valid.
- **Voice file wins on conflict.** Macros that previously declared tone now defer to the voice file. Inline tone declarations forbidden.
- **Personalization is an add-on layer.** Same folder reserves space for future audience profiles, brand assets, etc.
- **Modules are physical folders.** Each gets its own README, strategy, templates, failure log. Future video/article slots reserved with stub READMEs.
- **No new content templates beyond migrated Thread_Template.** Originals workflow, hands-on review, reply templates explicitly out of scope per user direction.
