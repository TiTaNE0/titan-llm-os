## 2026-04-19: Core Plugins - Vault Structure & Architecture
The vault uses a decentralized hub-and-spoke architecture where global logic (00_Templates, 01_Projects, 02_Tasks, 03_Brain, 04_Logs, 99_Archive) lives in core folders and project execution happens in specific nodes. Key components: Project Passports (mission/tech stack/constraints), flat task directory with YAML frontmatter, System_Agents.md as OS Kernel with global rules and synthetic macros like /new_task, daily session logs for context restoration, and 99_Archive for stale tasks. Critical handshake mechanism: AGENTS.md file acts as bootloader commanding agent to follow symlink into vault and load System_Agents.md rules before work begins. Core plugins include Kanban for visual boards, Templater for dynamic task generation, and implied Dataview for YAML frontmatter optimization.

## 2026-04-19: Different Agents Leverage - Tool Agnostic Architecture
The architecture's "secret sauce" is decoupling the "Brain" (Obsidian) from the "Executioner" (OpenCode/Codex/Cursor), making the system tool-agnostic. Key insights: Intelligence is locked in Markdown files, not AI chat history; new tools can access the brain via .vault_link symlink and AGENTS.md loader; most modern agents can use System_Agents.md as system prompt or rules file; built-in RAG in IDEs like Codex/Cursor can index the vault through symlink for intelligent suggestions. Two caveats: tool-specific triggers (OpenCode uses /commands, others may prefer natural language) and symlink support in cloud-based IDEs. The architecture creates a "Portable Developer Personality" - a Manual for My Brain that makes the user the architect and AI a replaceable worker reading blueprints.
